# Nginx Proxy Manager

เครื่องมือโอเพนซอร์สที่ออกแบบมาเพื่อช่วยจัดการและกำหนดค่า reverse proxy โดยใช้ Nginx ผ่านอินเทอร์เฟซเว็บที่ใช้งานง่าย โดยไม่จำเป็นต้องมีความรู้เชิงลึกเกี่ยวกับการกำหนดค่า Nginx ด้วยตนเอง NPM ถูกพัฒนาขึ้นในรูปแบบ Docker container ทำให้ติดตั้งและใช้งานได้ง่าย เหมาะสำหรับทั้งผู้ใช้ทั่วไปและผู้ดูแลระบบที่ต้องการจัดการการเข้าถึงบริการเว็บหรือแอปพลิเคชันภายในเครือข่าย เช่น การโฮสต์ที่บ้านหรือในธุรกิจขนาดเล็ก

## Reverse Proxy คืออะไร?

Reverse proxy คือเซิร์ฟเวอร์ที่ทำหน้าที่เป็นตัวกลางระหว่างผู้ใช้ (client) และเซิร์ฟเวอร์ภายใน (backend servers) โดยรับคำขอจากผู้ใช้และส่งต่อไปยังเซิร์ฟเวอร์ที่เหมาะสม จากนั้นส่งคำตอบกลับไปยังผู้ใช้ การใช้ reverse proxy ช่วยเพิ่มความปลอดภัย ประสิทธิภาพ และความสามารถในการขยายระบบ (scalability) ได้ดังนี้:

- ความปลอดภัย: ปกป้องเซิร์ฟเวอร์ภายในโดยซ่อนที่อยู่ IP และใช้ SSL/TLS เพื่อเข้ารหัสการรับส่งข้อมูล
- การกระจายโหลด (Load Balancing): แบ่งการรับส่งข้อมูลไปยังเซิร์ฟเวอร์หลายเครื่องเพื่อป้องกันการทำงานหนักเกินไป
- การจัดการทราฟฟิก: สามารถกำหนดเส้นทางการรับส่งข้อมูลตาม URL หรือโปรโตคอล และแคชเนื้อหาเพื่อเพิ่มความเร็ว
- การเข้ารหัส SSL: ทำให้สามารถใช้งาน HTTPS ได้ง่ายขึ้นด้วยการจัดการใบรับรอง SSL

## คุณสมบัติหลัก

- มี GUI ที่ช่วยให้ผู้ใช้สามารถตั้งค่า proxy host, redirection, และ SSL ได้โดยไม่ต้องเขียนโค้ด Nginx ด้วยตนเอง
- รองรับการขอและต่ออายุใบรับรอง SSL ฟรีโดยอัตโนมัติ หรือสามารถใช้ใบรับรองที่กำหนดเองได้
- รองรับ HTTP, HTTPS, TCP, และ UDP รวมถึงการกำหนดค่า stream host สำหรับการส่งต่อพอร์ต
- รองรับหลายบัญชีผู้ใช้ การกำหนดบทบาท และการบันทึก log การเปลี่ยนแปลงเพื่อความปลอดภัยและการตรวจสอบ
- การทำ Load Balancing: สามารถกำหนดค่าเพื่อกระจายโหลดไปยังเซิร์ฟเวอร์หลายเครื่องในส่วนการตั้งค่าขั้นสูง
- ทำงานได้ดีในสภาพแวดล้อม Docker โดยสามารถเชื่อมต่อกับ container อื่นๆ ผ่าน Docker network เพื่อความปลอดภัยและประสิทธิภาพ
- มีตัวเลือก “Block Common Exploits” เพื่อป้องกันการโจมตีทางไซเบอร์ทั่วไป

## การติดตั้ง

<Steps>

### สร้างโฟลเดอร์สำหรับเก็บไฟล์

```bash
mkdir nginxproxymanager
cd nginxproxymanager
```

### สร้างไฟล์ docker-compose.yml

```bash
sudo nano docker-compose.yml
```

คัดลอกข้อความด้านล่างนี้ แล้วนำไปวางในไฟล์ `docker-compose.yml` ที่คุณเพิ่งสร้างขึ้นภายในโฟลเดอร์ `nginxproxymanager`

```yml filename="~/nginxproxymanager/docker-compose.yml"
services:
  app:
    container_name: nginx-proxy-manager
    image: jc21/nginx-proxy-manager:latest
    restart: unless-stopped
    ports:
      - 80:80
      - 81:81
      - 443:443
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
```

### บันทึกไฟล์

### รัน Docker Compose

หลังจากที่คุณได้สร้างไฟล์ `docker-compose.yml` แล้ว ให้รันคำสั่งด้านล่างเพื่อเริ่มต้น Nginx Proxy Manager

```bash
sudo docker compose up -d
```

คุณจะเห็นผลลัพธ์คล้ายกับตัวอย่างด้านล่าง ซึ่งแสดงว่า Container กำลังถูกดาวน์โหลดและเริ่มต้น

```bash
[+] Running 36/36
 ✔ app Pulled                                                                     27.7s
[+] Running 2/2
 ✔ Network nginxproxymanager_default  Create...                                     0.1s
 ✔ Container nginxproxymanager-app-1  Start...                                     0.5s
```

### ตรวจสอบสถานะของ Container

คุณสามารถตรวจสอบสถานะการทำงานของ Container ได้ด้วยคำสั่ง

```bash
docker ps
```

หาก Nginx Proxy Manager ทำงานอยู่ คุณจะเห็นผลลัพธ์คล้ายกับตัวอย่างนี้

